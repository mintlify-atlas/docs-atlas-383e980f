The entire Open Brain data model is a single table: `thoughts`. It stores the raw text of each thought, its semantic embedding, LLM-extracted metadata, and a content fingerprint used to deduplicate writes.

## The `thoughts` table

```sql
create table thoughts (
  id uuid default gen_random_uuid() primary key,
  content text not null,
  embedding vector(1536),
  metadata jsonb default '{}'::jsonb,
  content_fingerprint text,
  created_at timestamptz default now(),
  updated_at timestamptz default now()
);
```

### Columns

<ResponseField name="id" type="uuid" required>
  Primary key. Auto-generated with `gen_random_uuid()` on insert.
</ResponseField>

<ResponseField name="content" type="text" required>
  The raw text of the captured thought. Not null. This is the canonical source of truth — the embedding and metadata are derived from it.
</ResponseField>

<ResponseField name="embedding" type="vector(1536)">
  A 1536-dimensional float array representing the semantic meaning of `content`. Generated by `openai/text-embedding-3-small` via OpenRouter. Used for cosine similarity search via the HNSW index.
</ResponseField>

<ResponseField name="metadata" type="jsonb">
  Structured metadata extracted by `openai/gpt-4o-mini`. Defaults to `{}`. See [metadata structure](#metadata-jsonb-structure) below.

  <Expandable title="properties">
    <ResponseField name="type" type="string">
      Thought classification. One of: `observation`, `task`, `idea`, `reference`, `person_note`.
    </ResponseField>
    <ResponseField name="topics" type="string[]">
      Array of 1–3 short topic tags. Always at least one element.
    </ResponseField>
    <ResponseField name="people" type="string[]">
      Names of people mentioned in the thought. Empty array if none.
    </ResponseField>
    <ResponseField name="action_items" type="string[]">
      Implied to-do items extracted from the thought. Empty array if none.
    </ResponseField>
    <ResponseField name="dates_mentioned" type="string[]">
      Dates mentioned in the thought, normalized to `YYYY-MM-DD`. Empty array if none.
    </ResponseField>
    <ResponseField name="source" type="string">
      Set to `"mcp"` by the MCP server on every capture.
    </ResponseField>
  </Expandable>
</ResponseField>

<ResponseField name="content_fingerprint" type="text">
  SHA-256 hash of the normalized content (lowercased, trimmed, whitespace-collapsed). Used to detect and deduplicate identical thoughts. Computed inside the `upsert_thought` SQL function.
</ResponseField>

<ResponseField name="created_at" type="timestamptz">
  Timestamp of first insert. Defaults to `now()`.
</ResponseField>

<ResponseField name="updated_at" type="timestamptz">
  Timestamp of last update. Automatically refreshed by the `thoughts_updated_at` trigger on every `UPDATE`.
</ResponseField>

## Metadata JSONB structure

The `metadata` column holds the output of the LLM extraction prompt. Every thought captured through the MCP server will have at minimum `type` and `topics`.

```json
{
  "type": "task",
  "topics": ["product", "launch"],
  "people": ["Sarah", "Marcus"],
  "action_items": ["Follow up with Marcus about timeline"],
  "dates_mentioned": ["2026-03-15"],
  "source": "mcp"
}
```

| Field | Type | Notes |
|---|---|---|
| `type` | string | One of: `observation`, `task`, `idea`, `reference`, `person_note` |
| `topics` | string[] | 1–3 short tags. Always present. |
| `people` | string[] | Names mentioned. Empty if none. |
| `action_items` | string[] | Implied to-dos. Empty if none. |
| `dates_mentioned` | string[] | Dates in `YYYY-MM-DD`. Empty if none. |
| `source` | string | `"mcp"` for all MCP captures. |

<Note>
  The metadata fields are populated by a best-effort LLM extraction. The `embedding` column is what powers semantic search — it works regardless of how accurately the LLM classifies the type or extracts people. Metadata is a bonus layer for structured filtering on top.
</Note>

## Indexes

Three indexes are created on the `thoughts` table:

```sql
-- HNSW index for fast approximate nearest-neighbor vector search
create index on thoughts
  using hnsw (embedding vector_cosine_ops);

-- GIN index for fast JSONB containment queries (@>)
create index on thoughts using gin (metadata);

-- B-tree index for date range queries and ordering
create index on thoughts (created_at desc);
```

| Index | Type | Column | Purpose |
|---|---|---|---|
| HNSW (cosine) | HNSW | `embedding` | Powers `match_thoughts` semantic search |
| GIN | GIN | `metadata` | Powers `list_thoughts` filters (`type`, `topics`, `people`) |
| created_at desc | B-tree | `created_at` | Powers `list_thoughts` time range and ordering |

The HNSW index uses `vector_cosine_ops` because similarity is computed as cosine distance (`<=>`). This is the correct operator class for `openai/text-embedding-3-small` embeddings.

## Deduplication trigger

A trigger automatically keeps `updated_at` current on every row update:

```sql
create or replace function update_updated_at()
returns trigger as $$
begin
  new.updated_at = now();
  return new;
end;
$$ language plpgsql;

create trigger thoughts_updated_at
  before update on thoughts
  for each row
  execute function update_updated_at();
```

A partial unique index on `content_fingerprint` prevents duplicate rows:

```sql
create unique index idx_thoughts_fingerprint
  on thoughts (content_fingerprint)
  where content_fingerprint is not null;
```

## `match_thoughts` function

The semantic search function. Called by the `search_thoughts` MCP tool via `supabase.rpc("match_thoughts", ...)`.

```sql
create or replace function match_thoughts(
  query_embedding vector(1536),
  match_threshold float default 0.7,
  match_count int default 10,
  filter jsonb default '{}'::jsonb
)
returns table (
  id uuid,
  content text,
  metadata jsonb,
  similarity float,
  created_at timestamptz
)
language plpgsql
as $$
begin
  return query
  select
    t.id,
    t.content,
    t.metadata,
    1 - (t.embedding <=> query_embedding) as similarity,
    t.created_at
  from thoughts t
  where 1 - (t.embedding <=> query_embedding) > match_threshold
    and (filter = '{}'::jsonb or t.metadata @> filter)
  order by t.embedding <=> query_embedding
  limit match_count;
end;
$$;
```

### Parameters

<ParamField path="query_embedding" type="vector(1536)" required>
  The embedding of the search query. Must be 1536 dimensions to match the stored embeddings. Generated by calling OpenRouter with `openai/text-embedding-3-small`.
</ParamField>

<ParamField path="match_threshold" type="float" default="0.7">
  Minimum cosine similarity score (0.0–1.0) for a result to be included. Rows with `1 - (embedding <=> query_embedding) <= match_threshold` are excluded. The `search_thoughts` MCP tool overrides this to `0.5` by default for broader recall.
</ParamField>

<ParamField path="match_count" type="int" default="10">
  Maximum number of results to return. Applied after threshold filtering.
</ParamField>

<ParamField path="filter" type="jsonb" default="{}">
  Optional JSONB containment filter applied to the `metadata` column. An empty object `{}` disables filtering. Example: `{"type": "task"}` returns only task thoughts.
</ParamField>

### Return fields

<ResponseField name="id" type="uuid">
  The thought's primary key.
</ResponseField>

<ResponseField name="content" type="text">
  The raw thought text.
</ResponseField>

<ResponseField name="metadata" type="jsonb">
  The full metadata object for the thought.
</ResponseField>

<ResponseField name="similarity" type="float">
  Cosine similarity score between the query embedding and this row's embedding. Computed as `1 - (embedding <=> query_embedding)`. Ranges from 0.0 (no similarity) to 1.0 (identical).
</ResponseField>

<ResponseField name="created_at" type="timestamptz">
  When the thought was first captured.
</ResponseField>

## `upsert_thought` function

The write function. Called by the `capture_thought` MCP tool via `supabase.rpc("upsert_thought", ...)`. Inserts new thoughts or merges metadata on duplicates.

```sql
create or replace function upsert_thought(p_content text, p_payload jsonb default '{}')
returns jsonb as $$
declare
  v_fingerprint text;
  v_result jsonb;
  v_id uuid;
begin
  v_fingerprint := encode(sha256(convert_to(
    lower(trim(regexp_replace(p_content, '\s+', ' ', 'g'))),
    'UTF8'
  )), 'hex');

  insert into thoughts (content, content_fingerprint, metadata)
  values (p_content, v_fingerprint, coalesce(p_payload->'metadata', '{}'::jsonb))
  on conflict (content_fingerprint) where content_fingerprint is not null do update
  set updated_at = now(),
      metadata = thoughts.metadata || coalesce(excluded.metadata, '{}'::jsonb)
  returning id into v_id;

  v_result := jsonb_build_object('id', v_id, 'fingerprint', v_fingerprint);
  return v_result;
end;
$$ language plpgsql;
```

### Parameters

<ParamField path="p_content" type="text" required>
  The thought text to store. Used to compute the SHA-256 fingerprint after normalization: lowercase, trimmed, and whitespace-collapsed.
</ParamField>

<ParamField path="p_payload" type="jsonb" default="{}">
  A JSONB object containing a `metadata` key with the extracted metadata. Merged with existing metadata on conflict via the `||` JSONB concatenation operator.

  <Expandable title="expected shape">
    ```json
    {
      "metadata": {
        "type": "task",
        "topics": ["planning"],
        "people": [],
        "action_items": [],
        "dates_mentioned": [],
        "source": "mcp"
      }
    }
    ```
  </Expandable>
</ParamField>

### Return fields

<ResponseField name="id" type="uuid">
  The UUID of the inserted or updated row.
</ResponseField>

<ResponseField name="fingerprint" type="text">
  The computed SHA-256 hex string for the normalized content. Can be used to confirm deduplication — the same content always produces the same fingerprint.
</ResponseField>

<Warning>
  Do not alter or drop existing columns on the `thoughts` table. Extensions, recipes, and integrations depend on the stable schema. Adding new columns is safe; removing or renaming existing ones will break the MCP server and any community-built tooling.
</Warning>
